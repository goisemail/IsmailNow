Plan in short
- Objective: Build IsmailNow as a React Native app with offline-first local storage and Google Drive CSV backups for online storage. No backend API changes are planned initially.
- Data approach: Offline-first local database; Google Drive used for CSV backups as an online companion.
- Backend: No immediate changes; backend discussion postponed until needed.
- iOS builds: Local on macOS using Xcode; deploy directly to a connected iPhone.
- MVP offline database baseline: SQLite (react-native-sqlite-storage).
 - MVP Database choice: SQLite (react-native-sqlite-storage) chosen for offline-first MVP baseline.
 - Rationale: simple setup, strong community support, easy migrations, predictable performance for core features.

- Phase 3: UI/UX and core features (habits, history, reminders, navigation).
- Phase 4: Google Drive backup flow (CSV export and upload to Drive).
- Phase 5: iOS deployment workflow (local signing, provisioning, and device install).
- Phase 6: Testing and iteration (on-device testing, offline behavior, backup flow sanity checks).
